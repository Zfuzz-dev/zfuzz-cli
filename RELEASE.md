# Releasing @zfuzz/cli (with provenance + binary attestation)

Publishing moved from a laptop to GitHub Actions. **npm provenance can only be
generated by a trusted CI runner via OIDC** — it cannot be produced locally. Each
release now ships:

- **npm provenance** on `@zfuzz/cli` and all 8 `@zfuzz/cli-<platform>` packages
  (the "provenance" badge on npmjs.com, linking package → commit → CI run).
- **Sigstore build-provenance attestations** on every native binary
  (`actions/attest-build-provenance`), verifiable with `gh attestation verify`.
- A **pinned `checksums.json`** in `@zfuzz/cli`; the launcher refuses to execute a
  binary whose SHA-256 doesn't match (see `zfuzz-cli/scripts/verify-binary.js`).

The engine source stays private. Binaries are cross-compiled privately and only
cross into the public world as **GitHub Release assets** — never as source.

## One-time setup

1. Repo `Zfuzz-dev/zfuzz-cli` is **public**, Actions enabled.
2. Add repo secret **`NPM_TOKEN`** — an npm **Granular Access Token** scoped to the
   `@zfuzz` packages with publish rights. (Provenance needs the publish to run in
   CI; 2FA-publish is satisfied by the automation token.)

## Cutting a release

1. **Bump the version** everywhere to the new `X.Y.Z` (keep them in lockstep):
   - `zfuzz-cli/package.json` + its 8 `optionalDependencies` pins
   - the 8 `*/package.json`
   - `Zfuzz-dev/zfuzz-mcp` → `package.json` `version` + its `@zfuzz/cli` dependency pin
2. **Build the 8 binaries privately** from the engine, named exactly:
   `zfuzz-darwin-arm64`, `zfuzz-darwin-x64`, `zfuzz-linux-x64-gnu`,
   `zfuzz-linux-arm64-gnu`, `zfuzz-linux-x64-musl`, `zfuzz-linux-arm64-musl`,
   `zfuzz-win32-x64-msvc.exe`, `zfuzz-win32-arm64-msvc.exe`.
   (Keep the existing anti-leak build flags: `--remap-path-prefix`, trimmed paths.)
3. **Create a GitHub Release** on `Zfuzz-dev/zfuzz-cli`, tag `vX.Y.Z`, attach the 8
   binary assets. Publishing the release triggers `.github/workflows/release.yml`:
   download assets → `tools/stage-binaries.mjs` → `tools/gen-checksums.mjs` →
   `npm test` → sigstore attest → `npm publish --provenance` for all 9 packages.
4. **Release `@zfuzz/mcp`**: create a `vX.Y.Z` release on `Zfuzz-dev/zfuzz-mcp`
   (its workflow publishes the pure-JS wrapper with provenance).

## Verify (after CI is green)

```sh
npm view @zfuzz/cli dist-tags             # new version is latest
npm view @zfuzz/cli --json | grep -A3 provenance   # provenance present
gh attestation verify <binary> --repo Zfuzz-dev/zfuzz-cli   # sigstore chain
```

npmjs.com shows a **"Provenance"** section on each package page. Then pin the
directory install configs to `@zfuzz/mcp@X.Y.Z` and resubmit.

## Local dev

Set `ZFUZZ_BIN=/path/to/zfuzz` to run a self-built binary — this path bypasses the
integrity check by design. The published install path is always verified.
